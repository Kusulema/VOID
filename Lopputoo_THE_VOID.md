## 1. Введение
THE VOID — студенческий проект: веб-приложение интернет-магазина с простым, расширяемым кодом на PHP, оформленным по паттерну MVC. Цель — создать работоспособный прототип с возможностью администрирования контента и управления заказами.

## 2. Цели и задачи
- Реализовать удобный каталог товаров с категориями.
- Обеспечить регистрацию и авторизацию пользователей.
- Реализовать корзину и оформление заказа.
- Создать административную панель с CRUD для товаров, категорий и комментариев.
- Подготовить базовые тесты и описать меры безопасности.

## 3. Техническое задание (обзор)
Функциональные требования для проекта VOID (реализация в кодовой базе):
- Просмотр категорий и списка товаров (файлы: `view/catproducts.php`, `view/allproducts.php`).
- Страница детали товара: `view/productDetail.php`.
- Регистрация/вход: формы `view/formRegister.php`, `view/formLogin.php`.
- Корзина и оформление заказа: `view/cart.php` и логика в `controller/Controller.php`.
- Комментарии к товарам с модерацией: модель `model/Comments.php`, админ-страницы в `viewAdmin/commentsList.php`.
- Админ-панель: вход через `admin/index.php`, контроллеры в `admin/controllerAdmin/`, модели в `admin/modelAdmin/`.
- Сохранение медиа/картинок: папки `img/` и `images/`.

Дополнительные требования:
- Отправка почты через PHPMailer (папка `vendor/phpmailer/phpmailer`).
- Использование Bootstrap для UI (файлы в `public/css/` и `admin/public/css/`).

## 4. Архитектура системы
- Файловая структура: контроллеры — `controller/` и `admin/controllerAdmin/`, модели — `model/` и `admin/modelAdmin/`, представления — `view/` и `admin/viewAdmin/`, маршруты — `route/routing.php`, `routeAdmin/routingAdmin.php`.
- База данных: MySQL. Схемы и дампы: `clothing_shop.sql`, `newsportal.sql`.
- Конфигурация базы данных и подключение — `inc/Database.php`.
- Зависимости и автозагрузка — Composer (`vendor/autoload.php`).

Архитектурная диаграмма и скриншоты интерфейса рекомендуется добавить в финальную версию (маркер `[[screenshot:описание]]`).

## 5. Установка и запуск
1. Установите XAMPP (или другой веб-сервер с PHP и MySQL).
2. Скопируйте папку проекта в `htdocs`, например в `C:\xampp\htdocs\VOID5`.
3. Создайте базу данных (например, `void_db`) и импортируйте дамп `clothing_shop.sql`:

```sql
-- Пример команды в MySQL/MariaDB:
CREATE DATABASE void_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE void_db;
SOURCE path/to/clothing_shop.sql;
```

4. Настройте подключение в `inc/Database.php` (укажите `host`, `db`, `user`, `pass`).
5. (Опционально) Если в проекте используются зависимости Composer, выполните:

```bash
composer install
```

6. Откройте сайт в браузере: `http://localhost/VOID5` и админ-панель: `http://localhost/VOID5/admin`.

Примечание: при необходимости создайте файл `.env` или укажите параметры подключения в `inc/Database.php` согласно вашему окружению.

## 6. Тестирование
6.1. Типы тестов, реализованных/планируемых:
- Unit-тесты для моделей (папка `tests/unit/`).
- Integration-тесты для сценариев оформления заказа (папка `tests/integration/`).
- E2E/ручное тестирование — чек-листы в `tests/test-output*.txt`.

6.2. Как запускать тесты:
- Если настроен PHPUnit, запустить в корне проекта:

```bash
vendor/bin/phpunit --configuration phpunit.xml
```

- Для ручного тестирования используйте списки в `tests/test-output.txt` и `tests/test-output2.txt`.

6.3. Пример тест-кейса (Регистрация):
- Предусловия: сервер запущен, БД доступна.
- Шаги: открыть `formRegister.php`, заполнить форму, отправить.
- Ожидаемый результат: учётная запись создана, пользователь видит личный кабинет `account.php`.

## 7. Безопасность
7.1. Хранение паролей: использовать `password_hash()` и `password_verify()` (в проекте должна применяться эта логика при регистрации — проверьте `model/Register.php`).
7.2. Подготовленные запросы: все работы с БД через PDO и подготовленные выражения (`inc/Database.php`).
7.3. CSRF: добавлять CSRF‑токены в формы (рекомендуется реализовать, если ещё нет).
7.4. XSS: экранировать пользовательский вывод через `htmlspecialchars()`.
7.5. HTTPS: на продакшн-сервере обязателен.

## 8. Руководство пользователя (кратко)
- Регистрация: `view/formRegister.php` — заполните имя, e-mail, пароль.
- Вход: `view/formLogin.php`.
- Просмотр каталога: `view/allproducts.php` и `view/catproducts.php`.
- Добавление в корзину: кнопки на странице товара, переход в `view/cart.php`.
- Оформление заказа: заполнение адреса и контактных данных в форме заказа.

## 9. Руководство администратора
- Вход: `admin/index.php`.
- Управление товарами: `admin/viewAdmin/productList.php`, формы добавления/редактирования в `admin/viewAdmin/`.
- Модерация комментариев: `admin/viewAdmin/commentsList.php`.
- Управление пользователями и заказами — разделы в админке.
